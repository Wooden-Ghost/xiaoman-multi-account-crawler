# xiaoman-multi-account-crawler
# 小满官网多账号爬虫系统（当前项目配置已过期，无法复现，需要重新购买账号或开通付费服务）

一个基于 **PySide6 + Playwright** 的本地桌面工具，用于批量抓取小满平台中的公司信息，支持多账号配置、页码任务分配、运行状态查看、日志记录，以及 Excel 结果导出。

本项目围绕“批量采集 + 本地操作 + 结果整理”这一类工作流设计，适合需要按条件抓取公司数据、集中查看结果并进行后续整理的场景。

---

## 项目定位

该工具主要解决以下问题：

- 批量访问目标列表页并抓取公司信息
- 使用多个账号分配采集任务
- 为每个账号设置独立页码范围
- 提取列表页公司名、抽屉内公司名与官网信息
- 在本地界面中查看运行状态、统计信息和系统日志
- 导出原始数据、去重结果和统计信息，便于后续筛选、分析与继续处理

它适合的工作环境是：

- 本地桌面运行
- 多账号分批采集
- 人工监看运行过程
- Excel 结果输出与后续整理

## 主要功能

### 1. 多账号配置
- 支持 4 个账号分别加载 JSON 配置文件
- 支持手动选择配置文件
- 支持快速加载 `config` 目录下的全部账号文件
- 支持单独启用或停用某个账号任务

### 2. 任务分配
- 支持为每个账号设置起始页与结束页
- 支持按账号生成页码任务队列
- 支持设置 HS Code、国家代码、每页数量与输出目录

### 3. 页面采集
- 自动构造目标页面 URL
- 进入列表页后逐条抓取公司数据
- 提取以下字段：
  - 页码
  - 位置
  - 列表页公司名
  - 抽屉内公司名
  - 官网
  - 账号编号
  - HS Code
  - 国家代码
  - 爬取时间

### 4. 运行控制
- 支持开始爬取
- 支持暂停与继续
- 支持停止任务
- 支持实时查看当前账号、当前页码、成功数、失败数与有官网数

### 5. 验证码检测
- 支持检测常见验证码 / 滑块验证提示
- 运行过程中可提示人工处理验证环节

### 6. 数据查看
- 在图形界面中查看已抓取结果
- 支持查看系统日志
- 支持导出前检查当前结果数据

### 7. Excel 导出
导出结果文件时，自动生成多个工作表：

- 原始数据
- 合并去重数据
- 统计信息

官网字段会参与去重逻辑：
- 有官网的数据按 `website` 去重
- 无官网的数据保留

## 基本使用流程

### 第一步：准备账号配置文件
将账号 JSON 文件放入 `config/` 目录，或在界面中为每个账号手动选择配置文件。

### 第二步：设置采集参数
在主界面中设置：

- HS Code
- 国家代码
- 每页数量
- 输出目录

### 第三步：配置账号任务
为每个账号设置：

- 是否启用
- 配置文件
- 起始页
- 结束页

### 第四步：开始采集
点击“开始爬取”，程序会按任务队列执行采集。

### 第五步：查看运行状态
运行过程中可查看：

- 当前状态
- 当前账号
- 当前页码
- 成功数
- 失败数
- 有官网数
- 系统日志

### 第六步：查看结果并导出
采集完成后，可在数据查看页检查结果，并导出 Excel 文件。

---

## 输出内容

程序导出 Excel 文件时，通常包括以下工作表：

### `原始数据`
保存全部抓取记录。

### `合并去重数据`
按官网字段合并去重后的结果，用于后续筛选和整理。

### `统计信息`
包括：

- 总爬取条数
- 成功条数
- 失败条数
- 有官网条数
- 无官网条数
- 去重后总条数
- 去重后有官网条数
- 各账号爬取数
- 最早爬取时间
- 最晚爬取时间

## 已知限制

- 页面结构变化后，部分选择器需要同步调整
- Cookie 配置需要保持有效
- 验证码或滑块验证出现时，需要人工处理
- 当前版本以本地单机运行为主，适合本地流程化使用
- 导出结果以 Excel 为主，适合中小规模数据整理场景
